Lesson 3 Highlights

John Forrester

Highlight #1 Version Control

Version control is a unique system that allows users to upload files and see every version of that file throughout the development process. This allows for users to recall certain versions of a file or set of files whenever they desire. This can be extremely helpful when mistakes are made while working on certain projects or more recent versions of a file/multiple files are needed. Version control systems also allow for collaboration between multiple users. Recalling certain files can certainly create a more productive work environment for groups of people. It is important for IT majors to know about such services since correcting mistakes this quickly in our field can save us a great deal of time.

Highlight #2 Local Repository

Local Repositories contain all necessary files for a project, but at the local level. This means that local repositories are ideal for single users. Items from the local repository can be uploaded to a remote repository eventually for collaborative purposes. These repositories can sometimes also be referred to as private repoisitories. It's important for IT majors to understand how to use such platforms for solo work. These repositories allow for work on the fly that sometimes doesn't require connection to the internet. It is also possible to take information from remote repositories and download them to your own local repoisitory. The only downside to local repoisitories would be it's lack of collaborative potential.

Highlight #3 Remote Repository

Remote repositories are exactly what they suggest: they're repositories that are isolated directly from the users associated with them, but can be accessed just as easily as a local repository. Via the web or Local area network, these repositories can be used for efficient group work or projects. While being based on a network, people can edit files from any location with a connection. While this platform isn't as private as a Local repository, it can definitely be deemed as a more convenient resource to groups of people or sometimes just one person. It is important that IT members learn how to master the innerworkings and functions of remote repositories so that collaboration can be done more efficiently amongst people assigned to the same goal/project.

Highlight #4 GIT

GIT is a version control system that allows the editing and modification of files for projects. This can be done remotely or locally. GIT is desinged in such a way that makes version control simple and easy to utilize by the end user. The process is as simple as entering a couple of commands to actively recall, modify and relocate files. GIT is also one of the most secure version control systems known to the tech world. Since it is free and open source, there are many different versions available optimized for user convenience. It is important for IT majors to learn about GIT and know how to use it because data integrity and security is key to success in any IT field. Being able to collaborate work or recall information is vital in today's work environment.

Highlight #5 GIT Commands

GIT commands are used to properly utilize GIT and all of its functions. In a way GIT is almost like a a coding language since it uses pseudocode to execute certain actions. Commands such as GIT push and GIT pull for example can be used to gather and share data. GIT pull takes the data froma local repository and sends it to a local repository. This can be useful for updating your personal repository with changes made by other users. GIT push uploads your changed from a local repository to a remote repository. This is essential for sharing your ideas. It's a good idea for IT majors to understand GIT commands in order to use the system effectively. There are many resource online dedicated to teaching people how to properly use these commands.
